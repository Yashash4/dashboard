## Summary of Recommendations

1. **Input validation for provisioning parameters**: Add strict regex validation for `subdomain` (`/^[a-z0-9]([a-z0-9-]{0,61}[a-z0-9])?$/`), `ip` (validate as public IPv4), `sshUser` (alphanumeric only), and `email` (email format). This addresses C-01 and H-02.

2. **Sanitize username for htpasswd**: Enforce `dashboardUsername` to be alphanumeric-only (no shell metacharacters). This addresses C-02.

3. **Redact credentials from API responses**: The `full/route.ts` endpoint should return masked passwords (e.g., `****`) or require a separate, rate-limited endpoint for credential retrieval. This addresses C-03 and H-04.

4. **Add rate limiting to all admin endpoints**: Apply `rateLimitAsync()` to provisioning, credential access, bulk operations, and destructive actions. This addresses H-01.

5. **Enforce admin role in middleware for `/api/admin/*`**: Add admin role checking to middleware for API routes matching `/api/admin/` as a defense-in-depth measure. This addresses H-03.

6. **Encrypt customer API keys at rest**: Use `encryptField()` for `api_key` in `user_api_keys` table, consistent with SSH password encryption. This addresses M-05.

7. **Bind embedding service to 127.0.0.1**: Change the embedding server listen address from `0.0.0.0` to `127.0.0.1`. This addresses M-03.

8. **Fail loudly when encryption is not configured**: `encryptField()` should throw or log a warning when `isEncryptionConfigured()` returns false in production. This addresses H-05.
