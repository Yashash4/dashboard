 Context

 Mission Control is the premium Ultra-tier ($350/mo) feature for ClawHQ — an AI agent orchestration

 1B. Add task ownership validation on sub-resource routes

 Why: Comments, reviews, and activities routes accept a task_id in the URL path but never verify the
 task belongs to the authenticated user. A user who guesses another user's task ID can insert
 comments/reviews into their tasks.

 Files: comments/route.ts, reviews/route.ts, activities/route.ts
 Fix: Before any INSERT, verify the task exists and belongs to user:
 const { data: task } = await supabase
   .from("mc_tasks").select("id").eq("id", taskId).eq("user_id", user.id).single();
 if (!task) return NextResponse.json({ error: "Task not found" }, { status: 404 });

 1C. Fix assigned_agent_id: "none" bug

 Why: The create task dialog has <SelectItem value="none">None</SelectItem>. If user selects "None",
 the string "none" is sent to API. The API does assigned_agent_id || null but "none" is truthy, so it
  stores "none" as the agent ID. This breaks agent filtering, queue pickup, and task queries.

 Files:
 - src/app/api/mission-control/tasks/route.ts line 113 — add: assigned_agent_id: (assigned_agent_id
 && assigned_agent_id !== "none") ? assigned_agent_id : null
 - src/app/api/mission-control/tasks/[id]/route.ts — same fix in PATCH handler
 - src/components/mission-control/task-board.tsx line 411 — fix: assigned_agent_id: agentId &&
 agentId !== "none" ? agentId : null

 PHASE 2: CRITICAL BUG FIXES (must ship)

 2A. Replace hardcoded 94.5% system health

 Why: metrics/route.ts:67 has system_health_percent: 94.5 — a fake number shown to paying users as if
  it's real monitoring.

 File: src/app/api/mission-control/metrics/route.ts
 Fix: Compute composite health score from real MC data:
 - Agent availability (50% weight): (active agents / total agents) * 100
 - Session success rate (30% weight): from today's mc_sessions
 - Error ratio (20% weight): ((total events - error events) / total events) * 100 from last 24h
 mc_events

 Already querying mc_agent_status, mc_tasks, mc_sessions. Just add mc_events query for error severity
  count, then calculate weighted score.

 2B. Add SSE broadcast to heartbeat route

 Why: Heartbeat upserts mc_agent_status and inserts mc_events but never calls emitMCEvent(). The

 File: src/app/api/mission-control/agents/heartbeat/route.ts

 - emitMCEvent(user.id, "agent_status_changed", { agentId: agent_id }) after every upsert
 - emitMCEvent(user.id, "new_event", { ... }) after inserting state change event
